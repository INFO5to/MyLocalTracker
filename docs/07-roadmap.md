- base funcional del sistema

- vista de repartidor
- vista publica del cliente
- login interno

## Pendientes de alta prioridad

### 1. Salir de Twilio Sandbox

Objetivo:

- usar un sender real de empresa
- dejar de depender de limites de prueba

### 2. Pulido visual de segunda pasada

Objetivo:

- refinamiento de tipografia
- animaciones suaves
- mejor responsive en dashboard y tracking
- componentes mas consistentes

### 3. Robustez movil del tracking

Objetivo:

- validar distintos telefonos
- validar foreground prolongado
- pulir mensajes cuando se pierde senal

### 4. Seguridad avanzada para cliente

Opciones futuras:

- expiracion de links
- OTP
- ocultar mas datos del repartidor

## Backlog de producto

## Recomendacion de priorizacion

### Camino corto

1. Pulido visual fino
2. Salir de sandbox

### Camino de endurecimiento

1. Seguridad de links
2. Mejoras de tracking movil

## Definicion de exito de la siguiente etapa

Se considerara un siguiente gran avance cuando:

- el cliente vea un remitente real de negocio
- el link abra sin friccion desde cualquier celular
- el tracking siga sintiendose vivo
- el negocio pueda operar pedidos sin soporte tecnico
